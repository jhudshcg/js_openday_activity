Coding Challenge - Help

Getting started

work in pairs and make sure you swap who is doing the coding from time to time.

  1. Explore the existing code. The HTML, Javascript and the CSS. Can you guess what each part does?
  2. Test your predictions. Try modifying a small bit of code and then testing what it does. Make the change, save the file and then reload the page in the browser. (Remember you can undo changes with Ctrl-Z)
  3. In pairs answer the following questions:
    1. What does line 17 in script.js do?
    2. What would line 95 in sprite.js do?
    3. What file and line does the code to update the fps count start? (you'll need to explore the code)
    4. What file and line does the code to check collisions between the sprites and the containing box start?
    5. How would you change the border color of the outer box containing the sprites to blue?
  4. Can you get the sprite animation to start in a different direction? Discuss the changes you made.

The main challenge

Make the sprite bounce off all the sides of the box.

  1. Planning is the way to succeed. First solve the problem in a simplified form, and you can discuss in pairs to help each other.

    You'll start by making a flow chart. Open a new tab in your browser and go to https://draw.io (or click this link)

    Then open the spritecollission.svg file. Read it and then finish the flow chart.

    1. You will use this flowchart to help you fix the JS code! Now make sure you read the comments in the code...
    2. How do you get the X, Y coordinates of the sprite bounding box?
    3. How do you get the sprite direction?
    4. What is the condition that needs to be met to change the direction of the sprite?
    5. Study the code and fix it so the sprite bounces from all sides of the box!
  2. Evaluate your solution. What works well, what could be improved? Do you have all the information you need to make those improvements?
